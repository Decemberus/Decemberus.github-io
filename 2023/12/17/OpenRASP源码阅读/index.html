包含三种类型的checker，一是js插件检测，意味着这个checker会调用js plugin进行攻击检测，二是java本地检测，意味着是调用本地java代码进行攻击检测，三是安全基线检测，是用于检测一些高风险类的安全性基线检测，检测其配置是否有安全隐患。

而这些位移就是位掩码来在一个整数中独立开启或关闭多个标志位

比如这样

如果配置值是 3（二进制 11），那么表示启用了 SQL 和 COMMAND 两种检测。

具体到这段代码，位数代表了每种检测类型的唯一标识符。例如：

  • SQL 的位掩码是 1（二进制 0001）。
  • COMMAND 的位掩码是 1 << 1 或 2（二进制 0010）。
  • DIRECTORY 的位掩码是 1 << 2 或 4（二进制 0100）。

进入了这个方法以后，会进入我们CustomClassTransformer

在这个类中，能够进行字节码的修改，并且添加hook注解

addAnnotationHook()读取了com.baidu.openrasp.hook包中所有被@HookAnnotation注解的class，然后缓存到集合hooks中，判断该对象是否是AbstractClassHook的子类，如果是，就调用addHook方法，传入该对象和该类的名称两个参数，将该对象作为一个hook添加到HookHandler中。

接下来会从所有在CustomClassTransformer中加载的类里面找到匹配的hook类，然后判断类是否可以修改，并且排除lambda表达式生成的类，最后就可以进行hook已经加载的类了

如此一来，后续类加载通过com.baidu.openrasp.transformer.CustomClassTransformer#transform的时候，对其进行匹配，判断是否需要hook
